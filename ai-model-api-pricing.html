Cost vocabulary and worked estimate

What is billable?

  • Input tokens: system prompt, user content, retrieved text, and often prior history.
  • Output tokens: generated text; reasoning/intermediate tokens can affect agent products.
  • Cached input: reused prefix charged under provider-specific read/write/storage rules.
  • Batch: asynchronous processing that may trade latency for lower token price.
  • Tool charge: search, container, storage, image, audio, or retrieval price beyond text tokens.
  • RPM / TPM: requests/tokens per minute; quota can constrain a cheap model before monthly cost.

Worked example: GPT-4.1 row

10,000 requests × 1,800 input + 450 output, with 50% input cached:

billable input = 10,000 × 1,800 × (1 − 0.50) = 9.0M
cached input = 10,000 × 1,800 × 0.50 = 9.0M
output = 10,000 × 450 = 4.5M
cost = (9.0 × $2.00) + (9.0 × $0.50) + (4.5 × $8.00) = $58.50 / month

Token usage only: include retries, tools, storage, observability, tax, and a safety margin in a deployment budget.

Context, cache, and rate-limit decisions

Long documents

Estimate full document plus system prompt, retrieved context, requested output, and history. For corpus search/extraction, test retrieval/chunking against one giant prompt; it can be cheaper and easier to inspect.

Caching

Cache stable, repeated prefixes such as policy instructions or a shared document set. It can reduce recurring input cost but adds write/storage cost and provider constraints like TTL or minimum size.

Rate limits

Use the account’s live limits, not a blog screenshot. Model, tier, project, region, history, prompt size, and preview status change effective RPM/TPM. Load-test below quota with backoff.

Budget and reliability guardrails

  1. Use separate project/service keys and tags per environment and workload.
  2. Set spend alerts and a service-side budget/kill switch; dashboards can lag.
  3. Log model ID, region/endpoint, input/output/cache/tool usage, latency, retries, and request purpose—while minimizing sensitive retention.
  4. Use exponential backoff with jitter for retryable 429/5xx; cap attempts so an outage cannot multiply spend.
  5. Pin model versions when supported; maintain evaluation and rollback before deprecations.

Advanced cost traps and anti-patterns

Tokenizer, modality, and agent-loop accounting

Token counts differ by tokenizer and modality. Image, audio, video, search, file storage, code execution, and retrieved documents can have separate units. Agent loops generate intermediate input/reasoning and tool traffic; set maximum turns/tools per user task.

Priority/flex, region, and deprecation risk

Priority or flex/batch processing changes latency/capacity tradeoffs. Direct API, cloud-reseller, and regional endpoints can differ in price and availability. Aliases and preview models are not a capacity plan: record version, migration date, and fallback.

Anti-patterns:
  • Comparing prompt-only price while ignoring output, cached input, and tools.
  • Assuming a consumer subscription includes API use.
  • Mixing one provider’s premium tier with another’s base quota.
  • Using old screenshots instead of pricing/rate-limit pages.
  • Treating a published limit as guaranteed sustained throughput.
  • Sending an entire corpus because a model advertises a large context window.
