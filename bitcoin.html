Bitcoin Wallet Cheatsheet

A comprehensive guide to Bitcoin self-custody, wallet types, security practices, and advanced concepts for beginner to advanced users.

I. Introduction

Purpose, Scope & Disclaimer

Quick reference for understanding, choosing, using, and securing Bitcoin wallets, emphasizing self-custody. Covers fundamentals, types, operations, security, advanced topics, comparisons, and pitfalls.

Why Self-Custody Matters
  • Core Principle: Reinforces the mantra "Not your keys, not your coins." Control your private keys, control your Bitcoin. Third parties (exchanges) introduce counterparty risk.
  • Benefits: Financial sovereignty, censorship resistance.
  • Responsibility: Requires diligence in securing keys/seed phrase.
Disclaimer

Educational purposes only, not financial or security advice. Technology evolves. Always Do Your Own Research (DYOR).

II. Bitcoin Wallet Fundamentals

What is a Wallet?

An interface (software/hardware) managing private/public keys. Allows viewing balance, generating addresses, and signing transactions. Doesn't *store* coins, just keys.

Public vs. Private Keys

Private Key: Secret code to authorize spending (Keep Secret!). Public Key: Derives addresses, shareable. Relationship is one-way.

Bitcoin Addresses

Identifiers for receiving BTC. Formats: Legacy (1...), Nested SegWit (3...), Native SegWit (bc1q... - preferred), Taproot (bc1p... - newest).

Address Types & Best Practices
  • Legacy (P2PKH): Starts with 1. Highest fees, lowest efficiency. Widely compatible.
  • Nested SegWit (P2SH-P2WPKH): Starts with 3. Wrapper for SegWit, lower fees than Legacy. Good compatibility bridge.
  • Native SegWit (Bech32 / P2WPKH): Starts with bc1q. Lowest fees, best error detection. Standard for most modern wallets.
  • Taproot (Bech32m / P2TR): Starts with bc1p. Most recent. Enables advanced scripting/privacy features. Growing adoption.
  • Best Practice: Use a new address for each transaction received to enhance privacy. Most wallets handle this automatically. Avoid address reuse.
Seed Phrase (BIP-39)

12/24 word mnemonic representing the master private key. The ONLY backup for your funds! Defined by BIP-39.

Importance & Derivation
  • CRITICAL Importance: If your device fails, this is the *only* way to recover access to your Bitcoin on a new wallet.
  • Generation: Should be securely generated *by the wallet software/hardware* itself, ideally offline for cold storage.
  • Derivation: Uses standards like BIP-32 (HD Wallets) and path standards (BIP-44, BIP-84, BIP-86) to generate a nearly infinite tree of keys and addresses from this single seed phrase (+ optional passphrase).
  • Tool (Caution!): Ian Coleman BIP39 Tool (Use Offline ONLY! Never enter your real seed online).
How Transactions Work

Spend UTXOs (unspent inputs) to create new ones (outputs). Pay miners a fee (sat/vB) for confirmation. Transactions wait in the Mempool.

Transaction Lifecycle
  • UTXOs (Unspent Transaction Outputs): Individual "chunks" of bitcoin you own. Your balance is the sum of your UTXOs.
  • Inputs & Outputs: Sending involves selecting UTXOs (inputs), creating new UTXOs for the recipient (output) and potentially change back to yourself (output). Input UTXOs become spent.
  • Transaction Fees (sat/vB): Paid to miners based on transaction *size* in bytes, not value. Higher fee rate = higher priority. Critical during network congestion.
  • Mempool: A "waiting room" for unconfirmed transactions broadcast to the network. Miners select transactions, usually prioritizing higher fee rates.
  • Confirmations: Inclusion in a valid block added to the blockchain. 6+ confirmations often considered highly secure.
  • Key Link: mempool.space (Visualize mempool, fees, blocks, transactions)

III.A Hot Wallets (Online)

Software Wallets (Apps)

Applications on computers/phones. Keys exposed to online threats. Good for frequent, smaller amounts. Distinguish Custodial vs. Non-Custodial.

Custodial vs. Non-Custodial
  • Non-Custodial: You hold the seed/keys. Full control & responsibility. True self-custody.
  • Custodial: Third party holds keys (e.g., exchange wallet). Trust required. Not self-custody.
Examples (Non-Custodial):
  • Desktop: Electrum, Sparrow Wallet, Specter Desktop
  • Mobile: BlueWallet, Muun Wallet, Samourai Wallet (*note legal issues*), Electrum Mobile
Pros & Cons
  • Pros: Convenient, often free, quick transactions.
  • Cons: Keys vulnerable to device compromise (malware, phishing, theft). Not for significant long-term holdings.
Web Wallets

Accessed via web browser. Often custodial. Generally discouraged for significant value due to browser/website/phishing risks.

III.B Cold Wallets (Offline)

Hardware Wallets

Purpose-built physical devices securing keys offline. Signatures done on-device. Highest security against online threats.

Pros & Cons
  • Pros: Best protection from malware/hacking, strong physical security features (PIN, tamper-evidence), recovery via seed.
  • Cons: Cost, requires physical device for signing, potential physical loss/theft (mitigated by seed backup).
Examples & Links:
  • Trezor (Model T, Safe 3)
  • Coldcard (Mk4, Q1)
  • Ledger (Nano S Plus, Nano X, Stax)
  • BitBox02 (Bitcoin-only Edition)
  • Seedsigner (DIY)
  • Blockstream Jade
  • Foundation Passport

(See Section VII for detailed comparison)

Paper Wallets

Printing private/public keys on paper. Generally obsolete and unsafe due to spending complexity and physical risks.

Risks
  • Difficult/unsafe partial spending (often requires sweeping entire balance, exposing keys).
  • Vulnerable to physical damage (fire, water, fading).
  • Requires extremely secure offline generation process.
  • Easily lost, stolen, or accidentally discovered.
  • Compared to hardware wallets, offers poor usability and less robust security for practical use.
Air-Gapped Setups

Using an offline device (PC, Pi, specific HW wallets) that never connects to internet. Transactions via SD/QR (PSBTs). Extremely high online security.

Pros & Cons
  • Pros: Maximum protection against online threats (malware, network attacks).
  • Cons: Requires technical setup/knowledge, more steps per transaction (transferring PSBTs), relies on secure handling of transfer media (SD card).
Examples:

Coldcard, Seedsigner used in air-gapped mode; dedicated offline computer running Electrum/Sparrow with watch-only wallet online.

IV. Core Wallet Operations

Wallet Creation & Setup

Generate seed phrase securely *on device*. Backup seed ACCURATELY & securely (offline, steel recommended). Set strong PIN/password. VERIFY backup!

Steps:
  1. Install Software / Initialize Hardware: Follow official instructions. Verify HW firmware if applicable.
  2. Generate Seed Phrase: Let wallet generate securely. Avoid pre-generated seeds.
  3. Backup Seed Phrase: Write down words accurately, in order (#1-12/24). Double check. Store securely offline immediately. Consider steel backups (Cryptosteel, SeedPlate).
  4. Verify Backup: Crucial! Re-enter words during setup or use device "check backup" feature *before* funding wallet significantly.
  5. Set PIN/Password: Strong, unique code for device/software access.
  6. Optional: Add BIP-39 Passphrase (See Advanced VI.A).
Receiving Bitcoin

Generate fresh address. Verify address on hardware wallet screen to prevent clipboard hijacking. Share address/QR with sender.

Sending Bitcoin

Enter recipient address (double/triple check!). Enter amount. Set fee (sat/vB). Review details. Confirm address/amount on hardware wallet screen.

Steps:
  1. Enter Recipient Address: Paste/scan. Check first/last characters carefully against intended address.
  2. Enter Amount.
  3. Set Transaction Fee (sat/vB): Check network conditions (mempool.space). Balance urgency and cost.
  4. Review Transaction Details: Check address, amount, fee on software screen.
  5. Confirm on Hardware Wallet: Verify address & amount again on HW trusted screen before physically confirming signature.
  6. Broadcast Transaction.
Checking Balance & History

Use wallet interface. Optionally use public block explorer (like mempool.space) with *addresses* (not keys!) - note privacy implications.

V. Security & Custody

Seed Phrase Security (CRITICAL)

Treat your seed phrase like the ultimate key to your vault. Protect it physically and digitally above all else.

RULE #1: NEVER EVER type, photograph, scan, or digitally input your seed phrase into ANY online or potentially compromised device (computer, phone, password manager, cloud storage, email, etc.). RULE #2: NEVER speak your seed phrase aloud where it could be recorded or overheard. RULE #3: NEVER give your seed phrase to ANYONE claiming to be support staff or offering help. Legitimate support NEVER needs your seed.
Storage Best Practices:
  • Use durable materials (paper minimum, steel recommended for fire/water resistance).
  • Protect from physical threats (fire, flood, corrosion, theft).
  • Store in secure, private locations (safe, hidden). Consider obscurity.
  • Redundancy: Consider multiple secure locations or Shamir Backup (Advanced VI.G). Balance security vs. accessibility.
  • Verification: Ensure backup accuracy. Test your recovery plan periodically.
Device Security (Hardware Wallets)

Secure the physical device and its operation. Buy direct, use strong PIN, verify transactions on screen.

Best Practices:
  • Sourcing: Buy directly from manufacturer or authorized reseller. Avoid third-party marketplaces.
  • Tamper Evidence: Inspect packaging/device upon arrival.
  • PIN: Use a strong, unique PIN (more than 4 digits recommended).
  • Passphrase: Understand usage and backup implications (Advanced VI.A).
  • Firmware Updates: Keep updated, but verify notifications (check official site), have seed backup ready, consider brief wait unless critical vulnerability.
  • Trusted Display: Always verify addresses and transaction amounts on the device's screen before confirming. Key defense against PC malware.
Software & Network Security

Maintain good digital hygiene: Use reputable software, keep OS updated, be wary of phishing, avoid public Wi-Fi for sensitive ops.

Key Points:
  • Wallet Software: Download from official sources only. Verify signatures if possible. Prefer well-maintained, reputable, ideally open-source wallets.
  • OS Security: Keep computer/phone OS and security software updated.
  • Phishing Awareness: Be extremely skeptical of unsolicited requests/links related to your wallet or funds. Verify via official channels.
  • Network: Avoid sensitive operations (seed entry/recovery) on untrusted networks.
  • Clipboard Malware: Always double-check pasted addresses. Verify on HW screen if possible.
Physical Security

Protect hardware wallet from theft/tampering ("Evil Maid"). Protect seed backups from theft, damage, discovery.

Considerations:
  • Hardware Wallet Device: Don't leave unattended in insecure places. PIN protects against casual theft. Be aware of sophisticated physical attacks if device is stolen (SE helps).
  • Seed Phrase Backups: Treat like physical bearer assets (cash, gold). Think about who might find them, how to protect against fire/flood/theft.
Backup & Recovery

Seed phrase is the primary backup, not the device. Have a clear, tested recovery plan. Consider estate planning.

Planning Steps:
  • Know the Process: Understand how to use your seed phrase to restore your wallet on a compatible device/software.
  • Practice Recovery: Consider simulating recovery on a testnet wallet or with a small amount to ensure your backup works and you understand the steps *before* an emergency.
  • Estate Planning: Crucial for ensuring assets aren't lost upon incapacitation/death. Options include:
    • Clear instructions stored securely with trusted executor/lawyer.
    • Multisignature setups involving trusted parties (family, lawyer, service).
    • Specialized inheritance services (research carefully).
    • Examples: Casa Inheritance, Unchained Capital Inheritance Protocol

VI. Advanced Operations & Concepts

BIP-39 Passphrase ("25th Word")

Optional secret word/phrase added to seed creates separate hidden wallets. Enhances security/deniability. Forgetting = PERMANENT loss!

Mechanism & Use Cases
  • Adds a user-defined secret to the seed for key derivation. `Seed + Passphrase = Different Wallet`.
  • Use Cases: Plausible deniability (small amount in main wallet, large in passphrase wallet); adds security factor beyond physical seed backup.
CRITICAL Risks
  • Forgetting the exact passphrase (case/space sensitive) means PERMANENT loss of funds in that wallet. No recovery possible.
  • Must be backed up securely and separately, or perfectly memorized (risky).
  • Typing errors lead to different/empty wallets.
Multisignature (Multisig)

Requires M-of-N keys (e.g., 2-of-3) to sign. Enhances security, enables shared custody, robust backups.

Setup & Use Cases
  • Requires coordinator software (Sparrow, Specter, Electrum, Nunchuk, Unchained, Casa) to manage public keys and create/sign transactions (PSBTs).
  • Use Cases: Eliminates single points of failure (needs multiple device/location compromises); joint accounts; corporate treasury; inheritance planning.
  • Resource: Sparrow Wallet Multisig Guide
Connect to Full Node

Connect wallet to your own Bitcoin Core node for maximum privacy and trust minimization.

Why & How
  • Why: Avoid leaking addresses/transaction history to third-party servers; independently verify Bitcoin rules.
  • How: Run Bitcoin Core node (or node packages like Umbrel, Start9). Connect compatible wallets (Sparrow, Specter, Electrum+Server) to node's IP.
  • Resource: Node implementation options
Coin Control (UTXO Mgmt)

Manually select which UTXOs to spend. Enhances privacy, helps manage fees, allows labeling.

Benefits & Wallets
  • Benefits: Avoid linking unrelated UTXOs (privacy); strategically select UTXOs for fee optimization; label UTXOs by source.
  • Wallets: Electrum, Sparrow, Specter typically offer good coin control features.
PSBTs (BIP-174)

Partially Signed Bitcoin Transaction format. Standard for air-gapped signing and multisig coordination.

Use Cases & Link
  • Use Cases: Transfer unsigned transactions to offline signers (Coldcard, Seedsigner) via SD/QR; pass transactions between multiple signers in a multisig setup.
  • Key Link: BIP-174 Specification
RBF & CPFP (Fee Bumping)

RBF (BIP-125): Replace your stuck transaction with higher fee. CPFP: Spend unconfirmed *incoming* UTXO with high fee.

Purpose & Link
  • Purpose: Techniques to increase the effective fee rate of transactions stuck in the mempool due to low fees.
  • RBF Note: Requires initial transaction to signal RBF support (wallet feature).
  • Key Link: BIP-125 Specification (RBF)
Shamir Backup (SLIP-39)

Splits seed into M-of-N shares (e.g., 3-of-5). Increases redundancy/security vs. single backup point.

Pros, Cons & Link
  • Pros: Lose some shares without losing access; attacker needs multiple shares.
  • Cons: More complex setup/recovery; less widely supported than BIP-39 (Trezor T, specific tools).
  • Key Link: SLIP-39 Specification
Testnet

Parallel Bitcoin network with valueless coins. Safely test features, practice recovery, learn without risking real funds.

VII. Hardware Wallet Comparison

Key Comparison Factors

Evaluate HW wallets based on: Security Model (SE vs MCU), Source Code (Open/Closed), Bitcoin Focus, Air-Gap capability, UI/UX, Advanced Features, Price, Reputation.

Factor Details:
  • Security Model: Secure Element (dedicated chip, often closed) vs. Generic MCU (standard chip, allows open HW/FW, relies on software security).
  • Source Code: Open (auditable, transparent) vs. Closed (trust manufacturer). Applies to Firmware and sometimes Hardware design.
  • Bitcoin Focus: Reduces attack surface vs. Multi-Coin convenience.
  • Air-Gap: True air-gap (microSD, QR, NFC) vs. USB/Bluetooth connection required.
  • User Interface: Ease of use, screen type/size, buttons vs. touch.
  • Advanced Features: Support for Passphrases, Multisig (PSBT), Coin Control, Shamir, etc.
  • Price Point.
  • Reputation: Manufacturer history, community trust, security incident handling.
Trezor (T/Safe 3)

Pros: Open source history (T HW/FW, Safe 3 FW), user-friendly Suite, good passphrase/SLIP-39 (T) support, Safe 3 adds SE. Cons: Model T=MCU, Safe 3 SE=closed, not air-gapped.

trezor.io

Coldcard (Mk4/Q1)

Pros: Bitcoin-only, high security focus, true air-gap (SD/NFC/USB), excellent PSBT/multisig, open source FW, strong physical security. Cons: Steeper learning curve, needs coordinator SW, basic UI, Mk4=MCU.

coldcard.com

Ledger (Nano/Stax)

Pros: Uses SE, wide coin support, popular, beginner-friendly Ledger Live. Cons: Closed source FW/SE, "Recover" controversy, past data breach, relies on Ledger Live.

ledger.com

BitBox02 (BTC-Only)

Pros: Good security/usability (SE + open FW elements), minimalist, BTC-only option, good passphrase/multisig via app. Cons: Requires BitBoxApp, less air-gap flexibility.

bitbox.swiss

Seedsigner (DIY)

Pros: Low cost, DIY/educational, fully air-gapped (QR), stateless, fully open source HW/SW. Cons: Requires assembly/setup, usability varies, slower QR process.

seedsigner.com

Blockstream Jade

Pros: Affordable, Bitcoin-focused, optional stateless/blind oracle, QR air-gap, open source HW/FW. Cons: Oracle requires trust (optional), basic UI.

blockstream.com/jade/

Foundation Passport

Pros: Premium design, Bitcoin-focused, air-gapped (QR/microSD), security features, open source FW. Cons: Higher price, relies on app/coordinator.

foundationdevices.com

Which is Right for You?

Consider technical comfort, budget, altcoin needs, air-gap preference, open vs. closed source importance, required advanced features, and trust. No single "best" wallet exists; understand the trade-offs for your specific needs.

VIII. Common Issues & How People Lose Bitcoin

Seed Phrase Loss/Errors

Most common failure point! Physical loss, incorrect backup (words/order), backing up wrong seed, forgetting/losing passphrase.

Phishing / Social Engineering

Instant theft! Entering seed into fake sites/apps, tricked by fake "support" asking for seed.

Device/Software Issues

Forgetting PIN (needs seed recovery), accidental wipe without *tested* backup, rare software bugs.

Malware

Clipboard hijacking (check HW screen!), keyloggers (risk for SW wallets), fake wallet apps.

Physical Access / Theft

Seed backup found/stolen, unsecured HW stolen (PIN/passphrase are defenses), coercion, insider theft.

Supply Chain / Bad Wallets

Tampered device (rare for reputable vendors), using incompetent/malicious wallets.

Lack of Estate Planning

Funds become inaccessible upon owner's death/incapacitation.

IX. Glossary

Key Terms

Definitions for essential Bitcoin wallet terminology.

Address
Identifier used to receive Bitcoin (e.g., starts with 1, 3, bc1q, bc1p).
Air Gap
Keeping a device (signer) physically isolated from network connections.
BIP-32
Standard for Hierarchical Deterministic (HD) wallets, deriving keys from a master seed.
BIP-39
Standard for mnemonic seed phrases (12/24 words) representing the master seed.
BIP-44/84/86
Standards defining derivation paths for different address types (Legacy, SegWit, Taproot).
BIP-125
Replace-By-Fee (RBF) standard.
BIP-174
Partially Signed Bitcoin Transaction (PSBT) format standard.
Bitcoin Core
The primary, reference implementation of the Bitcoin full node software.
Block Explorer
Website to view blockchain data (transactions, addresses, blocks).
Blockchain
The distributed, public ledger recording all Bitcoin transactions.
Cold Storage
Storing private keys offline.
Coin Control
Manually selecting specific UTXOs for spending.
Confirmation
Inclusion of a transaction in a block on the blockchain.
CPFP
Child-Pays-For-Parent. Spending an unconfirmed input with a high fee to accelerate parent confirmation.
Custodial
A third party holds your private keys (e.g., an exchange).
Derivation Path
Instructions for deriving specific keys/addresses from the master seed (e.g., m/84'/0'/0'/0/0).
Electrum
Popular, feature-rich software wallet.
Fee Rate (sat/vB)
Transaction fee measured in satoshis per virtual byte.
Full Node
Software that independently validates all Bitcoin transactions and blocks.
Hardware Wallet
Physical device securing private keys offline.
HD Wallet
Hierarchical Deterministic wallet, derives keys from a single seed (BIP-32).
Hot Wallet
Wallet where keys are stored on an internet-connected device.
Keys
Private Key (secret, for signing) and Public Key (sharable, for receiving).
Ledger
Popular hardware wallet manufacturer.
Lightning Network
Layer-2 protocol for fast, cheap Bitcoin transactions (uses different wallet types/concepts).
Mempool
Pool of unconfirmed transactions waiting for miners.
Multisignature (Multisig)
Requires multiple keys (M-of-N) to authorize a transaction.
Non-Custodial
You control your own private keys/seed phrase.
Passphrase (BIP-39)
Optional secret word/phrase added to seed, creating a hidden wallet ("25th word").
Private Key
The secret data allowing spending of Bitcoin.
PSBT
Partially Signed Bitcoin Transaction format.
Public Key
Cryptographic key derived from private key, used to generate addresses.
QR Code
Square barcode often used to represent addresses or transaction data.
RBF
Replace-By-Fee. Bumping the fee of an unconfirmed transaction.
Satoshis (Sats)
Smallest unit of Bitcoin (1 BTC = 100,000,000 sats).
Secure Element (SE)
Tamper-resistant chip designed to securely store cryptographic keys.
Seed Phrase
12/24 word mnemonic backup for the master private key (BIP-39).
Shamir Backup (SLIP-39)
Splits seed into multiple shares, requiring a threshold for recovery.
Signature
Cryptographic proof generated using a private key to authorize a transaction.
Software Wallet
Wallet application running on a computer or phone.
Sparrow Wallet
Desktop software wallet focused on security and privacy.
Specter Desktop
Desktop software wallet focused on multisig and hardware wallet interaction.
Taproot
Recent Bitcoin protocol upgrade enhancing privacy and scripting capabilities (P2TR addresses).
Testnet
Parallel Bitcoin network for testing with valueless coins.
Trezor
Pioneering hardware wallet manufacturer.
UTXO
Unspent Transaction Output. An individual "chunk" of bitcoin.
Watch-Only Wallet
Wallet loaded with public keys/addresses to monitor balance/transactions, but cannot spend.
