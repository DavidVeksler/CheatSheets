.NET Ecosystem Cheatsheet

An interactive guide to modern .NET components, frameworks, and libraries for architects & senior developers.

.NET Platform Foundation

.NET SDK

The Software Development Kit providing tools to develop, build, test, and publish .NET applications. Includes the .NET CLI, compilers, and runtime.

Purpose

A comprehensive suite of tools essential for the .NET development lifecycle.

Key Components
  • .NET CLI (`dotnet`): Command-line interface for creating, restoring dependencies, building, running, testing, and publishing applications. Docs
  • Compilers: Roslyn for C# and VB.NET, F# compiler. Translates source code to CIL.
  • Runtime & Libraries: Includes the .NET Runtime and Base Class Library (BCL) necessary to run apps.
  • Build System (MSBuild): Handles the build process based on project files (`.csproj`, `.fsproj`). MSBuild Docs
Use Cases

Essential for any .NET development, from simple console apps to complex web services and cross-platform applications.

.NET Runtime (CoreCLR)

The execution environment for .NET applications. Manages memory (GC), JIT compilation, type safety, and exception handling.

Purpose

Provides the foundational services required to execute managed .NET code.

Core Components (CoreCLR)
  • JIT (Just-In-Time) Compilation: Converts CIL bytecode into native machine code at runtime. Managed Code
  • Garbage Collection (GC): Automatic memory management, reclaiming unused objects. GC Docs
  • Type System (CTS): Enforces type safety and defines how types are declared, used, and managed.
  • Exception Handling: Manages structured error handling.
  • Security: Provides mechanisms for code access security and verification.

Briefly, Mono is used for Xamarin/MAUI on iOS/Android, and NativeAOT compiles .NET to native code ahead-of-time. NativeAOT Docs

Base Class Library (BCL)

A rich set of core libraries providing fundamental functionalities like collections, I/O, networking, JSON handling, and threading. BCL Overview

Purpose

Provides ready-to-use types and utilities for common programming tasks, forming the bedrock of .NET applications.

Key Namespaces/Features
  • `System` Namespace: Core types (`string`, `int`, `DateTime`), I/O (`File`, `Stream`), collections (`List`, `Dictionary`), console operations. System Namespace
  • `System.Linq` Namespace: Language Integrated Query for querying objects, XML, and databases. LINQ Docs
  • `System.Text.Json` Namespace: High-performance JSON serialization and deserialization. System.Text.Json Docs
  • `System.Net.Http` Namespace: Modern HTTP client functionalities (`HttpClient`). HttpClient
  • `System.Threading.Tasks` Namespace: Asynchronous programming with `Task` and `Task` (async/await). Async Programming
CIL / MSIL

Common Intermediate Language (formerly MSIL). The platform-agnostic compiled output of .NET languages, executed by the CLR.

Purpose

Serves as an intermediate instruction set that can be JIT-compiled to native code on any platform supporting a .NET runtime. This enables language interoperability and platform independence. Managed Execution

Characteristics
  • CPU-Agnostic: Not tied to a specific processor architecture.
  • Verifiable: Can be verified for type safety by the CLR.
  • Contains Metadata: Includes information about types, members, and references.
Assemblies & Metadata

Units of deployment and versioning in .NET (DLLs or EXEs). Contain CIL code and self-describing metadata.

Purpose

Assemblies are the building blocks of .NET applications, packaging types and resources. Metadata allows the runtime to understand the types within an assembly without needing separate header files or an IDL. Assemblies in .NET

Key Aspects
  • Manifest: Contains assembly metadata (name, version, culture, referenced assemblies, security permissions).
  • Types: The CIL code for the types defined in the assembly.
  • Resources: Embedded resources like images or strings.
  • Scoping: Defines a boundary for type resolution and versioning.

Application Models & Frameworks

ASP.NET Core

Modern, cross-platform framework for building web apps, APIs, and microservices. High-performance and modular. ASP.NET Core Docs

Key Sub-Frameworks/Patterns
  • Minimal APIs: Lightweight syntax for creating HTTP APIs. Minimal APIs
  • MVC (Model-View-Controller): Pattern for building web UIs and APIs. MVC
  • Razor Pages: Page-centric model for building web UIs. Razor Pages
  • Blazor: Build interactive client-side web UIs with C#. (Blazor Server, WebAssembly, Hybrid). Blazor
  • Web APIs: Robust framework for building RESTful HTTP services.
Core Features

Middleware pipeline, routing, dependency injection, configuration, logging, Kestrel web server.

.NET MAUI

Multi-platform App UI. Build native mobile and desktop apps from a single C# codebase for iOS, Android, Windows, and macOS. .NET MAUI Docs

Purpose

Evolve Xamarin.Forms to provide a single project structure and unified API for creating native UIs across multiple platforms.

Key Aspects
  • Single Project: Manage code, resources, and platform-specific assets in one project.
  • Native UI: Renders native controls on each platform for authentic look and feel.
  • XAML and C#: Define UI using XAML markup or C# code-behind.
  • Access to Native APIs: Provides abstractions for common device features and allows direct platform API access.
  • Hot Reload: See XAML and C# code changes reflected live in running app.
Targets

iOS, Android, Windows (WinUI 3), macOS (via Mac Catalyst).

Key Libraries & Technologies

Entity Framework Core

Modern, cross-platform Object-Relational Mapper (ORM). Enables developers to work with databases using .NET objects. EF Core Docs

Key Features
  • LINQ to Entities: Query databases using Language Integrated Query.
  • Migrations: Manage database schema changes over time. Migrations
  • Database Providers: Supports SQL Server, SQLite, PostgreSQL, MySQL, Cosmos DB, and more. Providers
  • Change Tracking: Automatically tracks changes to entities for updates.
Use Cases

Simplifies data access logic in applications requiring interaction with relational or document databases.

Tooling & Ecosystem

IDEs

Integrated Development Environments for .NET development. Visual Studio, VS Code, and JetBrains Rider are popular choices.

Popular IDEs
  • Visual Studio: Full-featured IDE for Windows, also available for Mac. Extensive debugging, profiling, and project management tools. Visual Studio
  • Visual Studio Code (VS Code): Lightweight, extensible code editor with excellent .NET support via C# Dev Kit and other extensions. Cross-platform. VS Code
  • JetBrains Rider: Cross-platform .NET IDE known for its powerful code analysis, refactoring tools, and performance. Rider

Advanced & Architectural Concepts

Native AOT

Ahead-of-Time compilation that compiles .NET code directly to native machine code for faster startup, smaller deployments, and reduced memory. Native AOT Docs

Benefits
  • Faster Startup Time: No JIT compilation needed at runtime.
  • Smaller Deployments: Can produce self-contained executables with only necessary code.
  • Reduced Memory Footprint: Less overhead compared to JIT-compiled applications.
Trade-offs & Considerations
  • Longer Build Times: AOT compilation is more time-consuming.
  • Reflection Limitations: Dynamic features like reflection might require workarounds or be limited. Trimming/Reflection
  • Platform Specificity: Output is native to the target platform.
Use Cases

Serverless functions, CLI tools, containerized applications, scenarios where startup performance and deployment size are critical.

Key Considerations for Architects

Architectural Decision Points

When designing .NET systems, architects must weigh scalability (horizontal/vertical), resilience, security, maintainability, deployment strategies (containers, serverless, PaaS), and cost optimization. Choosing the right application models and libraries is crucial for success.

Core Tenets
  • Scalability: Design for growth. Will the system scale out (more instances) or scale up (more powerful instances)? How will data be partitioned or sharded if needed?
  • Resilience & Fault Tolerance: Implement patterns like retries, circuit breakers (e.g., using Polly), and health checks. Design for failure.
  • Security by Design: Incorporate security throughout the development lifecycle (authentication, authorization, data protection, secure coding practices). ASP.NET Core Security
  • Maintainability & Testability: Write clean, modular code. Employ DI, unit tests, and integration tests. .NET Testing
  • Deployment Strategies: Consider containers (Docker, Kubernetes), serverless (Azure Functions, AWS Lambda), Platform-as-a-Service (Azure App Service).
  • Cost Optimization: Monitor resource usage, choose appropriate cloud service tiers, and optimize for efficiency.
  • Observability: Implement robust logging, metrics, and tracing (e.g., using OpenTelemetry). OpenTelemetry with .NET

The choice of .NET frameworks and libraries should align with these architectural goals and the specific requirements of the project.
